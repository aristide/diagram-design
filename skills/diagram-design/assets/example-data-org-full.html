Data organisation · 5-tier storage pattern

Five progressive storage tiers — from raw source files through anonymization, staging, and aggregation to long-term archival. Each tier defines data quality, access rights, and file format standards.

TIER 1 · RAW

Source landing zone

  • Original files as received from data providers
  • All formats accepted: Excel, CSV, TXT, JSON, XML
  • Immutable — never modified after landing
  • Full audit trail for every ingestion event

Access: Data Engineers only

TIER 2 · BRONZE

Anonymization layer

  • Converted to Parquet for efficient querying
  • PII fields removed or pseudonymized per policy
  • Schema validated against registered metadata
  • Row-level lineage back to raw source preserved

Access: Engineers + Stewards

TIER 3 · SILVER

Validation & integration

  • Cleaned, deduplicated, and validated records
  • Standardized schemas across all sources
  • Linked via unique identifiers (entity resolution)
  • Quality scores and null-rate thresholds enforced

Access: All authenticated users

TIER 4 · GOLD

Publication-ready (focal)

  • Business-level aggregates ready for dissemination
  • CPI, GDP, trade, and labour market statistics
  • SLA-governed, tested, and documentation required
  • Consumed directly by Superset, APIs, and press briefs

Access: All (read-only)

TIER 5 · ARCHIVE

Long-term retention

  • Annual snapshots of Gold-tier datasets
  • Immutable — governed by retention policy
  • Cold storage: infrequent access expected
  • Required for regulatory compliance and audits

Access: Admins only
